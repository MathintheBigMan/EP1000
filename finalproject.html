Coding

For this section I will be explaining the important parts of the code that allow the logic of this game to work.
To make the code it is improtant to know the basic premis of the game. It is played by two players (usually on a piece of paper) and the play area consist of 9 boxes.
Within those 9 boxes each player will take turns to place down either a cross or a circle and the game ends when a line is formed by either player.
Now that we know how the game works, it's time to translate it into logic of code. To emulate the 9 box play area I used a 3x3 matrix to give the code a base and outline of the playarea. The zeros in the matrix represents an empty space.
Next we set an input funtion to let players to select the box the would like to put thier "piece". To tanslate that into code, I made a curser system to act like a pointer to "point" too what each player will place their piece on.
Next I made a funtion to check when a player has "won" this means if either player's piece make up a row. The first checking seq are to check if either row or columb is made up of '1' or '2' representing either player.
The next checking seq is to check the diagonals of the matrix to see if either player 1 or 2 fills a diagonal. To make it easier to check the diagonal the program first checks if the middle cell is filled by either player. This makes it much faster when checking for a winner.
